git config --global -> para que la config sea globlal

git config --global core.autocrlf true -> para el Carriage Return y Line Feed de Windows 

______Comandos_____
ls -> para ver archivos y carpetas
pwd -> donde estamos trabajando
cd -> ingresar a los directorios
cd .. -> para volver al directorio anterior
mkdir -> crear un nuevo directorio
rm -> eliminamos el archivo
mv <file> <new-name-file> -> para cambiar el nombre de los archivos


____GIT____
git init -> inicializar un repositorio en git
git status -> vemos el estado de los archivos
git restore <file> -> para restaurar el archivo eliminado (incluso antes del commit)


____ENV____
Si yo quiero ignorar archivos o carpetas, puedo ignorarlas creando un archivo llamado .gitignore y nombrando los archivos que queremos ignorar. 
Luego deberemos commitear el .gitignore

___GIT STATUS___
git status -m -> tenemos la informacion mas ordenada
M(rojo) -> archivo Modificado
M(verde) -> archivo Modificado en la etapa de staged
?? -> no ha sido agregado
A(verde) -> agregando archivo al staged


___GIT DIFF___
Podemos ver los cambios del archivo y con la letra "q" salimos del diff
git diff --staged -> solo los cambios que estan en stage y no todo


___GIT LOG___
Para ver el historia podemos usar git log --oneline -> veremos un codigo con los commit


___BRANCH___
Creamos ramas para desarrollar una funcionalidad (independientemente de la rama principal) y luego solicitar realizar un merge a nuestra rama de main
git branch -> vemos en que rama estamos trabajando
git checkout -b nombreRama -> para crear una rama
cat <file> -> ver el archivo

Para nosotros poder llevar los cambios de la rama secundaria a la principal, debemos posicionarnos sobre la principal y escribir:
git merge nombreRama


__GITHUB__
Creamos el repositorio en github y copiamos las dos ultimas lineas:
-git remote add origin "........."
-git push -u origin main
Luego nos pide usuario y una key, la key tenemos que sacarla de los settings de github en Developer settings/Note/repo (all checked). Copiamos la llave y la pegamos 
en la terminal
